Dependencies

The components the pattern uses are gov.uk-heading-xl paired with inline radios with the descriptive text centrally aligned below each radio.

Background

When eliciting user sentiment, the benefit of the Likert scale is that they use a questions use a scale. This means people are not forced to express an either-or opinion, allowing them to be neutral should they so choose. The pattern follows the gov.uk guiding principle of one thing per page.

Use of the traditional vertical radio buttons creates a false hierarchy of content. User research which has shown that users react poorly to the traditional format of vertically stacked radios, which they interpret as the service ‘ranking’ them. Unlike other Government services where the user is required to answer as specifically as possible, this service is more nuanced. For example "I set myself targets when I have things to do, and usually meet them" doesn't have a binary answer.

By using the horizontal format of radios we can avoid the perception that the product is ‘ranking’ them. The reason the radios are inline with the statement is to fit with right-to-left mental model of how english-speaking users (whom our service is solely available to).

When to use

The Likert scale format should be used for a question or statement that requires a Likert scale response.

This should be used for questions or statements which are asking the user only one thing per page.

Research

Research findings

When we tested vertically stacked format of radio buttons, users interpret the response at the top as the most desirable choice, which therefore biases the user towards the ranking they believe to be ‘best’.

Theme: ‘Customers preferred the answer options presented horizontally for different reasons’
